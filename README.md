# Citizen Fraud Shield

**AI for Digital Public Safety** — ET AI Hackathon 2026, Topic 6

A hybrid AI system that detects digital arrest scams and fraud from a call transcript or message in seconds — before money moves, not after.

## The problem

Digital arrest scams — fraudsters impersonating CBI, ED, Customs, or Police — defrauded Indian citizens of over ₹1,776 crore in the first nine months of 2024 alone. These scams work by isolating victims mid-call: stay on video, don't tell your family, transfer money now. There's no point-of-contact tool that tells a citizen, in the moment, whether what they're hearing is real.

## What this does

Paste a suspicious call summary or message and get:
- A **verdict** (SCAM / LIKELY_SCAM / LIKELY_SAFE / SAFE) with a risk score
- The **specific red flags** detected, in plain language
- A **recommended action**
- An explainability breakdown of *how* the verdict was reached
- The same in **English, Hindi, Tamil, Telugu, Bengali, Marathi, or Kannada**

## Architecture

A two-layer hybrid pipeline, not a single LLM prompt:

1. **Rule-based pre-filter** — fast regex matching against five known digital-arrest-scam categories (authority impersonation, urgency/isolation, fabricated legal jeopardy, payment demands, fake official trappings)
2. **Groq (Llama 3.3 70B) reasoning layer** — receives the transcript *plus* the pre-filter's findings as context, and produces the final structured verdict

This makes the system explainable and auditable, not a black box — the "How this was detected" panel in the UI shows exactly which signals fed into the final call.

```
Citizen input
     │
     ▼
Rule-based pre-filter (regex, 5 categories)
     │
     ▼
Groq / Llama 3.3 70B reasoning layer
     │
     ▼
Structured verdict (risk score, red flags, explanation, action)
     │
     ├──▶ Citizen Fraud Shield UI (verdict, share-warning, report)
     └──▶ Live Intel dashboard (aggregate stats, session feed)
```

## Tech stack

- **Backend**: FastAPI, Groq API (`groq`, Llama 3.3 70B), Python 3.12
- **Frontend**: React, Vite, Tailwind CSS v4, lucide-react
- **Data**: 50-item synthetic transcript dataset (25 scam / 25 legit, including deliberately tricky legitimate messages to stress-test false positives)

## Key features

| Feature | Why it matters |
|---|---|
| Hybrid pre-filter + LLM pipeline | Explainable, not a black-box prompt wrapper |
| Live evaluation panel | Proves accuracy numbers live, in front of judges, instead of just asserting them in a deck |
| Multi-language responses | Directly addresses the hackathon's "language coverage" evaluation criterion |
| "Warn a family member" | Counters the isolation tactic scams rely on |
| Live intel dashboard | Same engine, second audience — demonstrates scalability into telecom/bank/NCRB use cases |

## Running locally

```bash
npm run setup   # one-time: creates backend venv, installs all dependencies
npm run dev     # runs backend (port 8000) and frontend (port 5173) together
```

Requires a Groq API key in `backend/.env` (copy from `backend/.env.example`). Get a free key at [console.groq.com](https://console.groq.com).

## Evaluation results

Run `python backend/evaluate.py` (with venv activated) to test against the full 50-item dataset and get accuracy / false-positive / false-negative rates. Results are also viewable live in the app itself via the "Run live evaluation" panel.
